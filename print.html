AI 日报

今日AI资讯

  1. Nano Banana Pro 动画新玩法： 介绍了一种利用 Nano Banana Pro 制作动画的方法。首先使用指令 "write \"A high-quality 3D medical illustration of a [whatever you want]\"" 来生成图像，例如“写‘动脉硬化的 3D 医学插图’”。接着，将生成的图片作为首帧图输入到 AI 视频工具（如 Kling）中，即可生成视频动画。此方法旨在快速、低成本地创建动画。

  2. GPT-5.2 Codex 处理长任务的优化： Numman Ali 分享了在 GPT-5.2 Codex 中处理长运行任务的经验。他指出，虽然该模型擅长长任务，但需要明确的指导以避免偏离目标。在 AGENTS.md 文件的顶部添加特定指令，可以帮助 Codex 连贯地执行长达数小时的任务。

  3. Kling 动态控制功能受关注： TechHalla 转发了关于 Kling 动态控制（Motion Control）功能的推文，称其“AI 已经失控了”，并提供了使用方法。该功能允许用户利用参考视频中的动作来生成图像。

  4. Gemini CLI 免费开放 Gemini 3 权限： Gemini CLI 已向所有免费账户用户开放了 Gemini 3 的使用权限。用户需要在 /settings 中启用“预览功能”来体验。

  5. Nano Banana Pro 城市海报生成器： Nano Banana Pro 推出了强大的城市海报生成功能。用户只需输入城市名称，AI 即可自动调整海报的艺术风格、3D 字体装饰、云海艺术风格、地理信息、天际线等元素，生成具有独特艺术风格和丰富信息的城市海报。

  6. NotebookLM 笔记集成 Gemini App： NotebookLM 的笔记现已可以作为上下文添加到 Gemini App 中。这意味着用户可以直接在 Gemini 中针对 NotebookLM 的内容进行提问，极大地增强了信息整合和利用的效率。

  7. Codex 正式支持 Skills 功能： Codex 工具现已正式支持 Skills 功能，该功能最初由 Claude 提出。Skills 允许用户将执行特定任务所需的指令、脚本和资源打包成可复用的模块，通过命令或自动调用来完成复杂任务。

  8. Nano Banana Pro 提示词价值凸显： 一则推文介绍了 Nano Banana Pro 的一个高价值提示词，用于解决 NotebookLM 自动生成的幻灯片无法编辑的问题。该提示词旨在生成具备特定风格但内容可编辑的幻灯片。

  9. Kling 动态控制实现演技迁移： HAL2400 转发了关于 Kling 动态控制功能的推文，指出该技术可能打破演技对性别和年龄的限制，允许一个演员扮演多个角色，通过参考视频的动作来生成逼真内容。

  10. Kling 动态控制的视频生成能力： Justine Moore 对 Kling 的新动态控制工具赞不绝口，称其在AI 视频领域实现了突破。该工具能够处理复杂的动作，如体操动作，并支持唇语同步和镜头运动。

  11. GPT-5.2 Codex 的强大之处： Numman Ali 评价 GPT-5.2 Codex Extra High 是一个“方法论上的野兽”，能够持续更新 OpenAI Codex OAuth 插件，展现出强大的自主学习和构建能力，仿佛在为未来工作打下基础。

  12. Kling 2.6 动画生成示例： OscarAI 分享了使用 Kling 2.6 生成的**《Hellsing》**（《地狱之歌》）角色战斗场景。通过结合 Image to Video 功能和精心设计的提示词，成功捕捉了战斗的速度感和动态感。

  13. ChatGPT 应用教程： 一篇推文提供了详细的教程，指导用户如何打开、连接和使用 ChatGPT 应用。

  14. ChatGPT 集成 Apple Music： ChatGPT 应用商店已上线，并且Apple Music 已接入。用户可以在 ChatGPT 中通过语音指令查找歌曲、创建歌单、试听音乐。

  15. ChatGPT 应用商店上线： ChatGPT 应用商店现已正式上线，用户可在个人设置中找到并访问。商店内已包含数十款应用。

  16. Kling 2.6 动作剪辑展示： WuxIA Rocks 分享了其使用 Kling 2.6 制作的动作剪辑 reel，展示了该工具强大的动态生成能力。

  17. Andrej Karpathy 的 2025 LLM 年度回顾： Andrej Karpathy 发布了关于 2025 年大语言模型 (LLM) 的年度回顾。他总结了六大变化：

    • 训练方法的范式升级：引入了 RLVR (Reinforcement Learning from Verifiable Rewards)，使模型能在有标准答案的环境中自主学习，提高了推理能力。
    • AI 智能的“形状”被理解：AI 的智能是**“参差不齐”（jagged intelligence）**的，在有“可验证奖励”的领域表现出色，而在常识、社交等领域仍有不足。
    • LLM 应用层浮出水面：Cursor 等应用证明了“LLM 应用”这一新物种的存在，它们负责上下文工程、模型编排和专业场景界面。
    • AI 搬进电脑：Claude Code 等产品展示了本地化 AI 智能体的潜力，强调了与开发者并肩工作的优势。
    • Vibe Coding 起飞：纯英文描述即可生成程序的**“vibe coding”**模式普及，降低了编程门槛，改变了软件形态。
    • 大模型的“图形界面时代”：Gemini Nano Banana 等产品预示着大模型将采用更符合人类喜好的图形化交互方式（如图片、动画）。

  18. GPT-5.2 Codex 处理大型代码库： Haider 评价 GPT-5.2 Codex 在处理大型、混乱的代码库时表现稳定，能够长时间运行而不偏离轨道，且输出质量高，尽管耗时较长。

  19. Kling 动态控制的惊艳效果： Dave Clark 对 Kling 的动态控制功能表示惊叹，称其在首次测试中展现了惊人的效果。

  20. ChatGPT 个性化设定更新： ChatGPT 新增了个性化设定，用户可以调节语气温暖度、热情程度、标题列表数量和表情符号使用频率。

  21. Codex Skills 功能详解： 一则推文解释了 Skills 的概念，它是一个包含说明书、代码脚本和资源的文件夹，用于教会 Codex 完成特定任务，如写代码、修复错误等。

  22. OpenAI 宣布 Codex 支持 Skills： OpenAI 正式宣布 Codex 工具支持 Skills 功能，使用户能打包和调用可复用技能，以执行特定任务。

  23. NotebookLM 升级至 Gemini 3： NotebookLM 的基础模型已升级为 Gemini 3，显著提升了其推理和多模态能力。

  24. Nano Banana Pro 的范式转换意义： 阿兹特克小羊驼 认为 Nano Banana Pro 是今年最重要的“范式转换”型产品，其在视觉元素处理方面表现突出，尽管在“工程图”等领域仍有不足，但它预示着AI 绘图行业的底层玩法正在发生不可逆的突变。

  25. Codex 解决 Plan 模式的痛点： weishu 指出，Codex 通过 Skills 机制解决了长期存在的Plan 模式不支持的问题，用户可以通过 $plan 命令或 execplan 来调用规划能力。

  26. Gemini MacOS 桌面应用功能设想： Josh Woodward 征集用户对Gemini MacOS 桌面应用的期望功能，旨在打造用户喜爱的产品。

  27. ChatGPT 原生邮件写作集成： ChatGPT 现已提供原生邮件写作集成，用户可以在聊天中直接更新、格式化文本，并方便地将邮件发送到邮件客户端。

  28. GPT-5.2 Codex 的卓越表现： javi 认为 GPT 5.2 Codex 的进步比 3.5 到 4 的升级更为显著。

  29. Kling 动态控制的强大能力： Stevie Mac 称赞 Kling 的动态控制功能解锁了惊人的能力，是模型和工具箱的重要补充。

  30. Gemini 3 Flash 的性能提升： Google 宣布 Gemini 3 Flash 性能达到前沿水平，速度是 2.5 Pro 的三倍。
